## 三、总轮次与总 Run 数（用于摊算成本）

- **有报告的 run 数**：31 次  
- **有报告的总轮次**：18+80+80+499+499+500+50+49+49+100+99+99+100+49+29+50+80+49+49+200+200+200+200+474+80+99+50+99+200+200+200 = **5 847** 轮  
- **失败但已消耗 token 的轮次**：约 180 轮（深度三国迷 500 轮 run 中途超时）  
- **总计入轮次**：约 **6 027** 轮（用于下面单轮成本估算）

每轮包含：  
- 1 次**测试 AI**（getNextIntent）：system + user prompt + 回复，max_tokens=2048；  
- 1 次**裁决 API**（callAdjudication）：system + user prompt + 叙事/JSON，max_tokens 约 512～900+400。  

每个 run 结束时还有 **1 次** 测试 AI 生成体验报告（getExperienceReport），max_tokens=1024。  
因此：  
- **总 LLM 调用次数** ≈ 6 027×2 + 31 ≈ **12 085** 次（仅按轮次粗算，未乘重试次数）。  
- 若存在解析失败重试、网络重试，实际调用次数会略高。
