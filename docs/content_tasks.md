## 内容 / 剧情协作待办

该文档用于协调“内容 AI”与“工程 AI”的分工，工程侧只需要从注册表读取数据，具体文案与剧情由内容侧填充。

### 1. Prompt / System 指令

- 文件：`src/agents/prompts.ts`
- 目标：
  1. 维护 System Prompt（古风旁白、合规边界）
  2. 依据用户分层（资深玩家 / 互动小说 / 轻度用户）增加差异化提示
- 要求：
  - 所有 Prompt 通过 `registerPrompt` 注册
  - 文本存储为常量，禁止分散在业务逻辑中

### 2. 事件模板

- 目录：`data/events/*.json`
- 规范：`EVENT_SPEC.md`
- 任务：
  - 维护既有事件（主线/支线/随机）并补充新剧情
  - 标注 `entry_conditions`、`resolution`、`cooldown_days`
  - 结合灰度测试反馈（新手引导、错误解释）新增辅助事件

### 3. 新手引导与 UI 文案

- 提交位置：后续将由内容侧提供 JSON（如 `data/ui/onboarding.json`），工程侧只做渲染
- 建议产出：
  - 引导脚本（3-5 条）：提示玩家输入示例
  - 错误解释模板：针对属性/状态不足的原因进行友好提示

### 4. 可交付方式

1. 在 `docs/content_tasks.md` 下方追加勾选列表，注明完成进度与文件路径
2. 新增/更新 JSON 或 Prompt 文件后，务必说明适用场景与触发条件
3. 如需工程配合（新增字段、渲染入口），在本文件中提出需求并 @ 工程 AI

> 工程 AI 只依赖接口层（`contentRegistry`）获取数据。内容侧可以持续提交 PR，仅修改文案/数据即可。
